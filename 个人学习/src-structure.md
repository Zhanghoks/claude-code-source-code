# Claude Code 源码目录结构详解

> 面向新手小白的保姆级指南

---

## 先来打个比方

想象 Claude Code 是一个**大型餐厅** 🍽️

- **厨房** = `src/` 源代码目录（做菜的地方）
- **服务员** = `entrypoints/` 负责接待客人（用户）
- **厨师长** = `query.ts` 核心引擎（统筹做菜流程）
- **各种厨具** = `tools/` 内置工具（切菜刀、炒锅等）
- **菜谱** = `commands.ts` 斜杠命令（告诉厨师要做什么菜）

---

## 目录一览

```
src/
├── main.tsx          ← 餐厅大门，进入程序的入口
├── query.ts          ← 厨师长，核心处理逻辑（最大最重要的文件）
├── QueryEngine.ts     ← 备用厨师长（SDK模式使用）
│
├── entrypoints/      ← 服务员入口（CLI/SDK/MCP三种接待方式）
├── commands.ts       ← 菜单本（所有斜杠命令）
├── tools.ts          ← 工具注册表（有哪些厨具可用）
├── Tool.ts           ← 工具定义（厨具标准规格）
│
├── tools/            ← 厨具仓库（40+种工具的具体实现）
├── commands/         ← 菜单做法详解（每个命令怎么执行）
├── components/       ← UI零件（屏幕上的按钮、对话框等）
├── screens/          ← 完整屏幕（整个页面）
│
├── services/         ← 后勤部门（API通信、日志分析等）
├── state/            ← 状态管理（当前营业状态）
├── hooks/            ← React钩子（ reusable 逻辑）
│
├── utils/            ← 通用工具箱（各地方都用的小工具）
├── types/            ← 类型定义（数据的"身份证"）
├── constants/        ← 常量（不会变的固定值）
│
├── bridge/           ← 外卖部门（和远程服务通信）
├── tasks/           ← 任务系统（派发给别人的工作）
├── context/          ← 上下文管理（记忆能力）
│
└── 其他...           ← 其他支持目录
```

## 核心文件详解

### 🚪 `main.tsx` — 餐厅大门

这是你运行 Claude Code 时第一个碰到的文件。类似于餐厅的**迎宾处**：
- 显示欢迎界面
- 初始化各种设置
- 把你带到正确的"座位"（REPL 交互界面）

**简单理解**：走进餐厅，第一个人跟你说"欢迎光临"。

---

### 👨‍🍳 `query.ts` — 厨师长（最重要！）

这是整个项目**最大的文件**（约 785KB），是核心中的核心。

**工作流程**：
```
你输入问题
    ↓
厨师长(query.ts)思考
    ↓
需要工具？ → 调用 tools/ 里的工具
    ↓
需要更多思考？ → 继续循环
    ↓
返回最终答案给你
```

**简单理解**：你是顾客，厨师长看完你的点单（query），然后指挥后厨做菜。

---

### 🔧 `tools/` — 厨具仓库

这里有 **40+ 种工具**，相当于厨房里的各种厨具：

| 工具 | 作用 | 类比 |
|------|------|------|
| Bash | 执行终端命令 | 电磁炉 |
| FileRead | 读取文件 | 读菜谱 |
| FileEdit | 编辑文件 | 调味 |
| Glob | 搜索文件 | 找食材 |
| Grep | 搜索内容 | 尝味道 |
| WebFetch | 获取网页 | 订购食材 |
| ... | ... | ... |

---

### 📋 `commands/` — 菜单做法

每个文件夹是一个斜杠命令的实现，比如：
- `/bash` — 执行命令
- `/read` — 读取文件
- `/grep` — 搜索内容
- `/web-search` — 网页搜索
- `/ask` — 提问
- `/task` — 任务管理
- ...总共 **80+ 个命令**

---

### 🖥️ `entrypoints/` — 服务员入口

三种不同的"接待方式"：

| 入口 | 作用 |
|------|------|
| `cli.tsx` | 普通用户使用的命令行界面 |
| `sdk/` | 开发者用的编程接口 |
| `mcp.ts` | MCP 协议服务器 |

**简单理解**：
- CLI = 顾客直接进店吃饭
- SDK = 外卖平台（别人帮你点餐）
- MCP = 第三方餐厅集成

---

### 🎨 `components/` — UI 零件

用 React + Ink 构建的终端 UI 组件，就像乐高积木：

- `Button` — 按钮
- `Text` — 文字
- `Box` — 容器
- `Message` — 消息气泡
- `Permission` — 权限提示框
- 等等...

---

### 🔌 `services/` — 后勤部门

支持性服务，不直接面对用户：

| 服务 | 作用 |
|------|------|
| `ApiClient` | 和 Claude API 通信 |
| `Analytics` | 上报使用数据 |
| `Compactor` | 压缩上下文（记忆整理） |
| `MCP` | MCP 协议支持 |
| `Plugins` | 插件管理 |

---

### 🌉 `bridge/` — 外卖部门

负责和**远程 Claude Code** 通信：
- 建立连接
- 处理认证
- 传输数据

---

### 📊 `state/` — 状态管理

整个应用的"营业状态"，用 React Context 管理：
- 当前权限设置
- 文件历史记录
- 各种开关状态

---

### 🧰 `utils/` — 通用工具箱

各种小工具函数，哪里都可能用到：
- 权限检查工具
- 消息处理工具
- Git 操作工具
- GitHub 操作工具
- 设置管理
- 等等...

## 新手学习建议

### 推荐阅读顺序：

1. **先看 `main.tsx`** — 了解程序入口
2. **再看 `query.ts`** — 理解核心循环（这部分最难）
3. **然后 `Tool.ts` + `tools.ts`** — 理解工具系统
4. **接着 `commands.ts`** — 理解命令系统
5. **最后逛 `components/`** — 看看 UI 长什么样

### 从简单开始：

- `components/` 里的文件较短，适合入门
- `utils/` 里的工具函数独立性好，容易理解
- 不要一开始就死磕 `query.ts`（太大太复杂）

## 常见问题

**Q: 为什么 `query.ts` 这么大？**
A: 因为整个 AI 对话循环都在里面，包括思考、决策、工具调用等核心逻辑。

**Q: 什么是"斜杠命令"？**
A: 就是 `/read`、`/bash` 这种以 `/` 开头的命令，类似于快捷指令。

**Q: Ink 是什么？**
A: 一个让你用 React 写终端 UI 的库，所以组件都是 `.tsx` 后缀。

**Q: MCP 是什么？**
A: Model Context Protocol，让 Claude 和外部工具通信的协议。

## 总结

Claude Code 的结构很清晰：

| 层级 | 核心文件 | 职责 |
|------|---------|------|
| 入口 | `main.tsx` | 迎接用户 |
| 核心 | `query.ts` | 思考决策 |
| 执行 | `tools/` | 具体做事 |
| 命令 | `commands.ts` | 用户指令 |
| 呈现 | `components/` | 界面展示 |

记住这个顺序，你就不会迷路！
