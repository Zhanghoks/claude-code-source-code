# Claude Code 搜索功能实现原理

> 面向新手的搜索系统详解

---

## 搜索功能有哪些？

Claude Code 有 **3 种搜索工具**：

| 工具 | 作用 | 类比 |
|------|------|------|
| `GlobTool` | 按文件名模式搜索 | 找文件（知道名字） |
| `GrepTool` | 按文件内容搜索 | 找文字（知道内容） |
| `WebSearchTool` | 网络搜索 | Google/百度 |

---

## 核心原理：复用 ripgrep

**重要发现**：GlobTool 和 GrepTool 底层都调用了 **ripgrep (rg)**！

这不是自己造轮子，而是复用系统自带的 ripgrep 工具，或者内置的 ripgrep。

---

## 文件搜索：GlobTool

### 工作流程

```
用户输入 glob 模式 (如 "*.ts")
        ↓
GlobTool.ts 接收参数
        ↓
调用 utils/glob.ts
        ↓
内部调用 ripgrep --glob
        ↓
返回匹配的文件列表
```

### 输入参数

```typescript
{
  pattern: "**/*.ts",    // glob 模式（必需）
  path?: "./src",        // 搜索目录（可选，默认当前目录）
}
```

### 示例

| 用户输入 | ripgrep 调用 |
|---------|-------------|
| `*.ts` | `rg --glob "*.ts"` |
| `**/*.js` | `rg --glob "**/*.js"` |
| `src/*.tsx` | `rg --glob "src/*.tsx"` |

---

## 内容搜索：GrepTool

### 工作流程

```
用户输入搜索内容 (如 "function hello")
        ↓
GrepTool.ts 接收参数
        ↓
调用 utils/ripgrep.ts
        ↓
执行 ripgrep 命令
        ↓
解析输出，返回结果
```

### 输入参数

```typescript
{
  pattern: "TODO",                    // 正则表达式（必需）
  path?: "./src",                    // 搜索目录
  glob?: "*.ts",                     // 文件名过滤
  output_mode?: "content",           // 输出模式
  "-B"?: 2,                         // 之前行数
  "-A"?: 2,                         // 之后行数
  "-C"?: 3,                         // 上下文行数
  "-n"?: true,                      // 显示行号
  "-i"?: true,                      // 忽略大小写
  head_limit?: 250,                 // 结果上限
}
```

### 三种输出模式

| 模式 | 说明 | 示例输出 |
|------|------|---------|
| `files_with_matches` | 只返回文件名 | `["a.ts", "b.ts"]` |
| `content` | 返回匹配行 | `["a.ts:1:hello", "b.ts:5:hello"]` |
| `count` | 返回每个文件的匹配数 | `[{file: "a.ts", count: 3}]` |

## 底层实现：ripgrep.ts

### ripgrep 三种模式

代码会自动选择合适的 ripgrep 模式：

```typescript
function getRipgrepConfig() {
  // 1. 系统 ripgrep（如果存在且用户允许）
  if (findExecutable('rg')) {
    return { mode: 'system', command: 'rg' }
  }

  // 2. 内置 ripgrep（Bun 内置）
  if (isInBundledMode()) {
    return { mode: 'embedded', command: process.execPath, argv0: 'rg' }
  }

  // 3. vendored ripgrep（自带备份）
  return { mode: 'builtin', command: '/path/to/rg.exe' }
}
```

### 执行流程

```
1. 构建命令参数
   ["--no-config", "-j", "1", "--glob", "*.ts", "pattern", "path"]

2. spawn 子进程
   spawn(rgPath, fullArgs, { signal: abortSignal })

3. 收集输出
   stdout.on('data', ...) → 累积到缓冲区

4. 处理结果
   - code 0 或 1 = 成功（0有匹配，1无匹配）
   - 其他 code = 错误

5. 超时处理
   - 默认 20 秒超时（WSL 是 60 秒）
   - 先发 SIGTERM，5秒后发 SIGKILL
```

### 安全特性

1. **路径 hijacking 防护**：使用命令名 `'rg'` 而非完整路径
2. **缓冲区限制**：stdout/stderr 最大 20MB
3. **资源限制**：单线程模式 `-j 1`（可配置重试）

## Glob.ts 工具函数

### 提取基础目录

```typescript
// 把 glob 模式拆成"基础目录" + "相对模式"
extractGlobBaseDirectory("src/**/*.ts")
// → { baseDir: "src", relativePattern: "**/*.ts" }
```

### 核心实现

```typescript
export async function glob(filePattern, cwd, { limit, offset }, abortSignal) {
  // 1. 处理绝对路径
  if (isAbsolute(filePattern)) {
    const { baseDir, relativePattern } = extractGlobBaseDirectory(filePattern)
    // 转成 ripgrep 能处理的相对路径
  }

  // 2. 构建 ripgrep 参数
  const args = [
    '--glob', pattern,      // glob 模式
    '-0',                   // null 分隔
    '--files',              // 只返回文件名
    '--', target            // 搜索目录
  ]

  // 3. 执行并解析结果
  const stdout = await execFilePromise('rg', args)
  const files = stdout.split('\0')

  // 4. 应用 limit/offset
  return files.slice(offset, offset + limit)
}
```

## GrepTool 额外功能

### VCS 目录自动排除

```typescript
// 这些目录自动不搜索（避免噪音）
const VCS_DIRECTORIES_TO_EXCLUDE = [
  '.git', '.svn', '.hg', '.bzr', '.jj', '.sl'
]
```

### 结果分页

```typescript
// 支持 offset + head_limit 分页
head_limit: 250,    // 最多返回 250 条
offset: 0,          // 从第几条开始

// 用户可以这样翻页：
// 第一页：offset=0, head_limit=250
// 第二页：offset=250, head_limit=250
```

### 默认限制

```typescript
const DEFAULT_HEAD_LIMIT = 250  // 未指定时默认 250 条

// 显式传 0 表示无限制
head_limit: 0  // 不要轻易用，会撑爆上下文
```

## 命令行等价

| Claude Code 调用 | 底层等价命令 |
|-----------------|-------------|
| `GlobTool("*.ts")` | `rg --glob "*.ts" --files` |
| `GrepTool("TODO")` | `rg "TODO"` |
| `GrepTool("TODO", path="src")` | `rg "TODO" src` |
| `GrepTool("TODO", glob="*.ts")` | `rg --glob "*.ts" "TODO"` |
| `GrepTool("TODO", output_mode="content")` | `rg "TODO" -n` |
| `GrepTool("TODO", "-C"=2)` | `rg "TODO" -C 2` |

## 总结

```
┌─────────────────────────────────────┐
│           用户/AI 调用              │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│      GlobTool / GrepTool           │  ← TypeScript 封装
│      tools/GlobTool/GlobTool.ts     │
│      tools/GrepTool/GrepTool.ts     │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│        glob.ts / ripgrep.ts         │  ← 工具函数
│        utils/glob.ts                │
│        utils/ripgrep.ts             │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│           ripgrep                   │  ← 底层搜索引擎
│      系统/内置/vendored              │
└─────────────────────────────────────┘
```

**核心思想**：用 TypeScript 封装 ripgrep，提供安全的接口、权限检查、结果格式化。
